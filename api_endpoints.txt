SECOM Failure Prediction API Endpoints
======================================

1. GET /
   - Description: Returns service metadata including project name, model type, default anomaly threshold, and evaluation metrics.
   - Query parameters: None
   - Response fields:
       • project (string)
       • model_type (string)
       • default_threshold (float)
       • metrics.precision_anomaly (float, optional)
       • metrics.recall_anomaly (float, optional)
       • metrics.f1_anomaly (float, optional)
       • metrics.accuracy (float, optional)
       • features (integer)

2. GET /health
   - Description: Health check endpoint used for platform monitoring.
   - Query parameters: None
   - Response fields:
       • status (string, expected value "ok")

3. POST /predict
   - Description: Detects anomalies for one or more SECOM samples using the pretrained autoencoder.
   - Headers: Content-Type: application/json
   - Request body:
       {
         "instances": [[float, ..., float], ...],  // Each inner array must have 558 numeric values.
         "threshold": float (optional)              // Overrides default 0.45 when provided.
       }
   - Response fields:
       • threshold (float)                         // Threshold used for the inference.
       • predictions (array):
           – reconstruction_error (float)
           – is_anomaly (boolean)
